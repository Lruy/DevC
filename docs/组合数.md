# 组合数、杨辉三角与二维前缀和

### 题目描述

组合数 $\binom{n}{m}$ 表示的是从 $n$ 个物品中选出 $m$ 个物品的方案数。举个例子，从 $(1,2,3)$ 三个物品中选择两个物品可以有 $(1,2),(1,3),(2,3)$ 这三种选择方法。根据组合数的定义，我们可以给出计算组合数 $\binom{n}{m}$ 的一般公式：

$$
\binom{n}{m} = \frac{n!}{m!(n-m)!}
$$

其中 $n! = 1 \times 2 \times \cdots \times n$；特别地，定义 $0! = 1$。

小葱想知道如果给定 $n,m$ 和 $k$，对于所有的 $0 \le i \le n,\; 0 \le j \le \min(i,m)$ 有多少对 $(i,j)$ 满足 $k \mid \binom{i}{j}$（即 $\binom{i}{j}$ 是 $k$ 的倍数）。

> 数据范围：$t \le 10^4$，$n,m \le 2000$，$k \le 21$。x

## 一、组合数：

### 1.1 什么是组合数

教室里有 $n$ 个同学，要选 $m$ 个去搬书——有多少种选法？这就是**组合数**要回答的问题。

形式化地说：从 $n$ 个**不同**物品中选出 $m$ 个，**顺序不重要**，方案数记为：

$$
\binom{n}{m} \quad \text{或} \quad C(n, m) \quad \text{或} \quad C_n^m
$$

### 1.2 一个具体例子

从 $(1,2,3)$ 三个物品中选两个，所有可能的选法：

$$
\underbrace{(1,2),\;(1,3),\;(2,3)}_{\text{共 3 种}} \quad\Rightarrow\quad \binom{3}{2} = 3
$$

注意：$(1,2)$ 和 $(2,1)$ 是**同一种**选法。

### 1.3 阶乘公式

$$
\binom{n}{m} = \frac{n!}{m!\,(n-m)!}
$$

其中 $n! = 1 \times 2 \times \cdots \times n$，特别定义 $0! = 1$。

### 1.4 特殊值


| 条件  | 值               | 含义                       |
| ----- | ---------------- | -------------------------- |
| $m=0$ | $\binom{n}{0}=1$ | 一个都不选，只有一种方式   |
| $m=n$ | $\binom{n}{n}=1$ | 全员选中，只有一种方式     |
| $m>n$ | $\binom{n}{m}=0$ | 不可能选出比总数还多的物品 |

### 1.5 对称性

$$
\binom{n}{m} = \binom{n}{n-m}
$$

> 选 $m$ 个拿走，和选 $n-m$ 个留下，方案数相同。

例：$\binom{5}{2} = 10 = \binom{5}{3}$（从 5 人中选 2 人搬书 = 选 3 人留下）

## 二、杨辉三角：组合数的"形状"

### 2.1 把组合数排成三角形

将所有 $\binom{n}{m}$（$0 \le m \le n$）按下表排列——第 $n$ 行写 $n+1$ 个数（$m=0,1,\dots,n$）：

```
n=0:              1
n=1:            1   1
n=2:          1   2   1
n=3:        1   3   3   1
n=4:      1   4   6   4   1
n=5:    1   5  10   10  5   1
n=6:  1   6  15  20   15  6   1
```

> 表格中的每个数，就是对应位置的组合数。例如第 5 行第 2 个数（从 0 开始编号）$=10$，正是 $\binom{5}{2}=10$。

### 2.2 核心递推关系

观察杨辉三角中一个数和它"肩上"两个数的关系：

```
        1    ← 上一行
       / \
      1   1
     / \ / \
    1   2   1   ← 2 = 1 + 1
   / \ / \ / \
  1   3   3   1  ← 3 = 1 + 2, 3 = 2 + 1
```

**规律**：每个数 = 左肩上的数 + 右肩上的数。

用组合数的语言写出来：

$$
\boxed{\binom{n}{m} = \binom{n-1}{m-1} + \binom{n-1}{m}}
$$

### 2.3 为什么这个递推是对的？

从 $n$ 个物品中选 $m$ 个。我们盯着**第 $n$ 个物品**看：

- **情况 A：不选第 $n$ 个物品** → 从前 $n-1$ 个中选 $m$ 个 → $\binom{n-1}{m}$
- **情况 B：选第 $n$ 个物品** → 从前 $n-1$ 个中再选 $m-1$ 个 → $\binom{n-1}{m-1}$

两种情况互斥且穷尽，所以相加：

$$
\binom{n}{m} = \binom{n-1}{m-1} + \binom{n-1}{m}
$$

#### 代数验证

上面用"选或不选"的组合意义给出了解释。我们也可以用阶乘公式做纯代数验证——把 $\binom{n-1}{m-1} + \binom{n-1}{m}$ 通分相加，看它是否等于 $\binom{n}{m}$：

$$
\begin{aligned}
\binom{n-1}{m-1} + \binom{n-1}{m}
&= \frac{(n-1)!}{(m-1)!\;[(n-1)-(m-1)]!} + \frac{(n-1)!}{m!\;[(n-1)-m]!} \\[8pt]
&= \frac{(n-1)!}{(m-1)!\;(n-m)!} + \frac{(n-1)!}{m!\;(n-m-1)!} \\[8pt]
&= \frac{(n-1)! \cdot m}{m!\;(n-m)!} + \frac{(n-1)! \cdot (n-m)}{m!\;(n-m)!} \\[8pt]
&= \frac{(n-1)! \cdot (m + n - m)}{m!\;(n-m)!} \\[8pt]
&= \frac{(n-1)! \cdot n}{m!\;(n-m)!} \\[8pt]
&= \frac{n!}{m!\;(n-m)!} \\[8pt]
&= \binom{n}{m}
\end{aligned}
$$

### 2.4 从数学公式到递推计算

至此，我们有了两种计算组合数的途径：


| 途径         | 公式                                           | 特点                       |
| ------------ | ---------------------------------------------- | -------------------------- |
| **阶乘公式** | $\binom{n}{m}=\dfrac{n!}{m!(n-m)!}$            | 需要乘除法，数太大容易溢出 |
| **递推公式** | $\binom{n}{m}=\binom{n-1}{m-1}+\binom{n-1}{m}$ | 只有加法，可以边算边取模   |

对于计算机来说，递推公式更加友好——尤其是组合数增长极快（$\binom{100}{50} \approx 10^{29}$，远超 `long long` 范围），但很多题目只关心**组合数模某个数 $k$ 的结果**。递推 + 边算边取模，完美契合。

## 三、动态规划：高效计算杨辉三角

### 3.1 DP 视角

把递推公式看成 **DP 状态转移方程**：

> `yh[i][j]` = $\binom{i}{j} \bmod k$（$0 \le j \le i$）
>
> 转移：`yh[i][j] = (yh[i-1][j-1] + yh[i-1][j]) % k`
>
> 边界：`yh[i][0] = yh[i][i] = 1 % k`

### 3.2 代码框架

```cpp
// yh[i][j] 表示 C(i, j) % k
for (int i = 0; i <= n; i++) {
    for (int j = 0; j <= i; j++) {
        if (j == 0 || j == i)
            yh[i][j] = 1 % k;   // 边界
        else
            yh[i][j] = (yh[i-1][j-1] + yh[i-1][j]) % k;  // 递推
    }
}
```

算法的时空复杂度均为 $O(n^2)$。对于 $n \le 2000$（如洛谷 P2822），$2000^2 = 4 \times 10^6$，轻松通过。

> 每个格子由它**正上方**和**左上方**的格子相加得到。

## 四、前缀和（一维）：回答区间查询的利器

### 4.1 问题引入

给定一个长度为 $n$ 的数组 $a[1 \dots n]$，有 $t$ 次询问，每次询问 $a[l \dots r]$ 的区间和。

- 朴素做法：每次从 $l$ 到 $r$ 累加 → 单次查询 $O(n)$，$t$ 次查询 $O(tn)$，太慢。
- 更优做法：**前缀和** → 预处理 $O(n)$，每次查询 $O(1)$。

### 4.2 定义

定义**前缀和数组** `pre[i]`：

$$
\text{pre}[i] = \sum_{x=1}^{i} a[x] = a[1] + a[2] + \cdots + a[i]
$$

> 习惯上令 `pre[0] = 0`（前 0 个元素的和）。

### 4.3 构建

递推构建，一次遍历即可：

$$
\text{pre}[i] = \text{pre}[i-1] + a[i]
$$

```
a:    [ 3,  1,  4,  1,  5,  9 ]
       ↓   ↓   ↓   ↓   ↓   ↓
pre: [0, 3,  4,  8,  9, 14, 23]
      ↑
    pre[0]=0
```

### 4.4 查询

区间 $[l, r]$ 的和 = 前 $r$ 项的和 - 前 $l-1$ 项的和：

$$
\boxed{\sum_{x=l}^{r} a[x] = \text{pre}[r] - \text{pre}[l-1]}
$$

```
查询 [3, 5]（即 a[3]+a[4]+a[5] = 4+1+5 = 10）

pre[5] - pre[2] = 14 - 4 = 10 
  ↑        ↑
前5项和  前2项和  → 差就是第3~5项的和
```

直观图示：

```
pre[5]:  [===============]           前 5 项
pre[2]:  [======]                    前 2 项
差:              [=======]        第 3~5 项
```

## 五、前缀和（二维）：从线到面

### 5.1 问题升级

现在有一个 $n \times m$ 的二维网格 $a[i][j]$（$1 \le i \le n,\; 1 \le j \le m$），有 $t$ 次询问，每次问一个矩形子区域的和：

$$
\sum_{x=x_1}^{x_2} \sum_{y=y_1}^{y_2} a[x][y]
$$

### 5.2 定义

定义二维前缀和 `pre[i][j]` 为**从 $(1,1)$ 到 $(i,j)$ 的矩形区域内所有元素之和**：

```
pre[i][j] = 以 (1,1) 为左上角、(i,j) 为右下角的矩形和
```

可视化（`pre[3][4]` 覆盖的区域）：

```
j=1  j=2  j=3  j=4  j=5
i=1: [x    x    x    x]   .
i=2: [x    x    x    x]   .
i=3: [x    x    x    x]   .
i=4:  .    .    .    .    .
```

### 5.3 构建：容斥原理

`pre[i][j]` 怎么算？考虑递推——把新格子 `a[i][j]` 加进来：

- **`pre[i-1][j]`**：覆盖了前 $i-1$ 行、前 $j$ 列（即第 $1$ 行到第 $i-1$ 行，第 $1$ 列到第 $j$ 列）的整个矩形。这个区域**包含了第 $j$ 列，但不包含第 $i$ 行**——`a[i][j]` 刚好在这个缺失的行里。
- **`pre[i][j-1]`**：覆盖了前 $i$ 行、前 $j-1$ 列（即第 $1$ 行到第 $i$ 行，第 $1$ 列到第 $j-1$ 列）的整个矩形。这个区域**包含了第 $i$ 行，但不包含第 $j$ 列**——`a[i][j]` 刚好在这个缺失的列里。

把这两个矩形加起来：

- 加的：前 $i-1$ 行 + 前 $j$ 列（A 区） + 前 $i$ 行 + 前 $j-1$ 列（B 区）
- **重叠部分**：前 $i-1$ 行、前 $j-1$ 列 被 A 和 B 各算了一次——被加了**两次**——这正是 `pre[i-1][j-1]`，需要减掉一次。
- **还缺什么**：`a[i][j]` 本身——它既不在 A（A 不含第 $i$ 行），也不在 B（B 不含第 $j$ 列），需要单独加上。

把上述分析写成公式：

$$
\boxed{\text{pre}[i][j] = \text{pre}[i-1][j] + \text{pre}[i][j-1] - \text{pre}[i-1][j-1] + a[i][j]}
$$

### 5.4 查询：容斥原理再现

查询以 $(x_1, y_1)$ 为左上角、$(x_2, y_2)$ 为右下角的矩形和：

$$
\boxed{\text{query} = \text{pre}[x_2][y_2] - \text{pre}[x_1-1][y_2] - \text{pre}[x_2][y_1-1] + \text{pre}[x_1-1][y_1-1]}
$$

### 5.5 一维 vs 二维对比


| 维度     | 构建                                                              | 查询                                                                  |
| -------- | ----------------------------------------------------------------- | --------------------------------------------------------------------- |
| **一维** | `pre[i] = pre[i-1] + a[i]`                                        | `sum[l..r] = pre[r] - pre[l-1]`                                       |
| **二维** | `pre[i][j] = pre[i-1][j] + pre[i][j-1] - pre[i-1][j-1] + a[i][j]` | `sum = pre[x2][y2] - pre[x1-1][y2] - pre[x2][y1-1] + pre[x1-1][y1-1]` |

## 六、洛谷 P2822 组合数问题

### 6.1 题目回顾

> 给定 $t$、$k$，接下来 $t$ 次询问，每次给 $n$、$m$。
>
> 求有多少对 $(i, j)$ 满足 $0 \le i \le n$，$0 \le j \le \min(i, m)$，且 $k \mid \binom{i}{j}$（即 $\binom{i}{j}$ 是 $k$ 的倍数）。

数据范围：$t \le 10^4$，$n, m \le 2000$，$k \le 21$。

### 6.2 思路拆解

这道题看似复杂，但可以拆成三个独立步骤：


| 步骤       | 做什么                         | 方法                          |
| ---------- | ------------------------------ | ----------------------------- |
| **第一步** | 算出所有$\binom{i}{j} \bmod k$ | 杨辉三角 + 递推（DP）         |
| **第二步** | 标记哪些位置$\bmod k = 0$      | 把余 0 的位置记为 1，否则为 0 |
| **第三步** | 快速回答矩形区域内有多少个 1   | 二维前缀和 →$O(1)$ 查询      |

> 步骤一和步骤二可以合并——在递推出 `yh[i][j]` 时直接判断是否整除。

### 6.3 第一步：杨辉三角（模 k）

```cpp
// yh[i][j] = C(i, j) % k （仅当 j <= i 时有意义）
for (int i = 0; i <= 2000; i++) {
    for (int j = 0; j <= i; j++) {
        if (j == 0 || j == i)
            yh[i][j] = 1 % k;
        else
            yh[i][j] = (yh[i-1][j-1] + yh[i-1][j]) % k;
    }
}
```

### 6.4 第二步 + 第三步：二维前缀和

定义 `pre[i][j]` = 在满足 $0 \le a \le i,\; 0 \le b \le j$ 的范围内，有多少对 $(a,b)$ 满足 $\binom{a}{b} \bmod k = 0$。

**每个格子的"原始值"**：

$$
a[i][j] = \begin{cases}
1 & \text{如果 } \binom{i}{j} \bmod k = 0 \text{ 且 } j \le i \\
0 & \text{否则}
\end{cases}
$$

**构建二维前缀和**（重点）：

对于 $j \le i$（在杨辉三角内）：

$$
\text{pre}[i][j] = \text{pre}[i-1][j] + \text{pre}[i][j-1] - \text{pre}[i-1][j-1] + (yh[i][j] == 0\; ?\; 1 : 0)
$$

对于 $j > i$（在杨辉三角外，C(i,j) 无定义，算作 0）：

$$
\text{pre}[i][j] = \text{pre}[i][i]
$$

> **为什么 `pre[i][j] = pre[i][i]`？**
>
> 第 $i$ 行只有 $0 \le j \le i$ 的位置才对应有效组合数。对于 $j > i$，查询区域扩展到这些列时不会增加任何新的"整除"计数——因为那些格子的值恒为 0。所以前缀和的值应该和 $j=i$ 时一致。

**查询答案**：

对于询问 $(n, m)$，答案就是 $\text{pre}[n][m]$， $j > i$ 时 `pre[i][j] = pre[i][i]` 。

### 6.5 边界初始化要点

二维前缀和的第 0 行和第 0 列需要单独初始化：

- **`pre[0][0]`**：看 $\binom{0}{0} \bmod k$ 是否为 0。$\binom{0}{0} = 1$，若 $k = 1$ 则 $1 \bmod 1 = 0$，`pre[0][0] = 1`；否则 `pre[0][0] = 0`。
- **第 0 行（`i=0`，$j \ge 1$）**：$\binom{0}{j} = 0$（$j>0$ 时无定义），所以不会贡献新的整除。因此 `pre[0][j] = pre[0][0]`（常数）。
- **第 0 列（`j=0`，$i \ge 1$）**：$\binom{i}{0} = 1$ 对所有 $i$ 成立。需要逐行累加：`pre[i][0] = pre[i-1][0] + (C(i,0) % k == 0)`。

### 6.6 算法复杂度


| 阶段           | 复杂度                                           |
| -------------- | ------------------------------------------------ |
| 杨辉三角递推   | $O(N^2)$，$N=2000$                               |
| 二维前缀和构建 | $O(N^2)$                                         |
| 每次查询       | $O(1)$                                           |
| **总计**       | $O(N^2 + t)$ ≈ $4 \times 10^6 + 10^4$，轻松通过 |

## 七、完整参考代码

```cpp
#include <iostream>
using namespace std;
typedef long long ll;

ll t, k;
ll yh[2005][2005];   // yh[i][j] = C(i, j) % k
ll pre[2005][2005];  // 二维前缀和：矩形 (0,0)~(i,j) 内 C%k==0 的个数

int main() {
    ios::sync_with_stdio(false), cin.tie(0);

    cin >> t >> k;

    // ============ 第一步：杨辉三角（模 k） ============
    for (int i = 0; i <= 2000; i++) {
        for (int j = 0; j <= i; j++) {
            if (j == 0 || j == i)
                yh[i][j] = 1 % k;
            else
                yh[i][j] = (yh[i-1][j-1] + yh[i-1][j]) % k;
        }
    }

    // ============ 第二步 + 第三步：二维前缀和 ============

    // 初始化 pre[0][0]
    pre[0][0] = (yh[0][0] % k == 0);  // C(0,0)=1, 检查是否被 k 整除

    // 初始化第 0 行和第 0 列
    for (int i = 1; i <= 2000; i++) {
        pre[0][i] = pre[0][0];                        // 第 0 行：只有列 0 有效，其余列值一致
        pre[i][0] = pre[i-1][0] + (yh[i][0] % k == 0); // 第 0 列：逐行累加
    }

    // 填充整个前缀和表
    for (int i = 1; i <= 2000; i++) {
        for (int j = 1; j <= 2000; j++) {
            if (j <= i) {
                // 标准二维前缀和公式（容斥）
                pre[i][j] = pre[i-1][j] + pre[i][j-1] - pre[i-1][j-1];
                if (yh[i][j] % k == 0) pre[i][j]++;  // 当前位置被 k 整除，额外 +1
            } else {
                // j > i：不在杨辉三角内，沿用 pre[i][i]
                pre[i][j] = pre[i][i];
            }
        }
    }

    // ============ 查询 ============
    while (t--) {
        ll n, m;
        cin >> n >> m;
        cout << pre[n][m] << '\n';  // O(1) 回答每次询问
    }

    return 0;
}
```
